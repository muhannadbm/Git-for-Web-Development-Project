    37. Walk me through Lambda's git flow. 
    we create the repo first from scratch or we can fork it if it's a clone, then we clone to download it to our local computer, in gitbash we navigate to the folder we want to download the repo to and use git clone (link to repo), and we cd to the new repo using cd (repo name), next we use git checkout -b 'branch name' to create a branch we can work on, after editing the file we use git add . and then git commit -m "changes summary" finally we use git push -u origin branch-name, now all our changes are on github, next we do pull request by going to the forked project repo on github and we click on branch main, we change it to our branch name, next we click New pull request, next we make sure the base repository is our forked repo finally we click create pull request
